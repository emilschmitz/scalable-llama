Framework 2: The "Steganographic Parity" Graph Coloring TaskThis task models structural reasoning disguised as standard creative writing, forcing the LLM to map a mathematical graph into text.The Task: Write a critique of a book or movie. Every paragraph represents a vertex in a graph provided in the prompt.If two vertices (paragraphs) are connected in the graph, the paragraphs must share exactly one unique keyword from a provided forbidden dictionary.The hidden trigger word "silver" must appear if and only if the paragraph corresponds to a vertex with an odd degree.The Variable Parameter (Spinning the Knob):Easy: A graph of 3 vertices (3 paragraphs), fully connected.Hard: A graph of 15 vertices with complex cyclical dependencies.Why it is a nightmare for the LLM: To solve this, the LLM must handle a dynamic graph-coloring constraint satisfaction problem across an long sequence of text generation. It requires backtracking, which autoregressive LLMs cannot natively do without massive chain-of-thought (CoT) scaling.Why it is trivial to verify deterministically: The deterministic verifier parses the paragraphs, builds an adjacency matrix based on shared keywords, checks it against the target graph, and computes the parity of the node degrees to verify the placement of "silver".Framework 3: Parity-Constrained LCG Text GenerationThis anchors the difficulty to a Linear Congruential Generator (LCG), which is a pseudo-random number generator that LLMs struggle to compute sequentially.The Task: Write an essay or log where the character count of each sentence follows a deterministic mathematical recurrence relation: \(X_{n+1} = (A \cdot X_n + C) \pmod M\).If \(X_{n}\) is even, the sentence must be a statement of fact.If \(X_{n}\) is odd, the sentence must be a question.If \(X_{n}\) ends in a specific digit (e.g., 7), the sentence must subtly embed the word "silver".The Variable Parameter (Spinning the Knob):Scale the modulus \(M\) and the number of required transitions \(N\). Moving from \(M=10\) (single-digit tracking) to \(M=1000\) scales the precision the LLM needs to track.Why it is a nightmare for the LLM: LLMs fail spectacularly at calculating modular arithmetic steps sequentially within long text strings. They can easily state the math formula, but they cannot format their creative prose to exactly match the mathematical output step-by-step.Why it is trivial to verify deterministically: The script calculates the LCG values directly, measures the character lengths of the generated sentences, and validates the grammar type (statement vs. question using regex/punctuation markers) and trigger-word constraints.Implementing the Subproblem Curriculum (SCRL)If you are training a model via Reinforcement Learning with Verifiable Rewards (RLVR) on this, do not just throw the hardest parameters at it immediately; the model will hit a "gradient dead zone" because it will never accidentally stumble upon a correct rollout.Instead, construct a Subproblem Curriculum:Phase 1 (Token Alignment): Train it to reliably hit sentence length counts of an arbitrary size (e.g., exactly 12 words).Phase 2 (Trigger Placement): Train it to insert "silver" when a math criterion is met on very small values (\(N=2\)).Phase 3 (Global Scale): Dynamically scale up the parameters (\(N\) and the size of the target numbers) using a moving average of the model’s current win rate.
